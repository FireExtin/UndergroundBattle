# 对战动作完整说明（Battle V4）

## 核心原则
- Battle 页面只保留主流程动作；调试专用动作继续留在 Debugger 页面。
- 先在牌桌点选卡牌，再在动作面板提交动作。
- 点击 `结束行动` 会自动提交 `pass_priority`；若本步骤结束，前端会自动补一次 `advance_phase`。
- 规则判定与结算以 Go 规则核返回为准。

## 主流程动作

### `build_asset`（建立资产）
- 用途：将一张手牌正面放入本方资产区，作为资产使用。
- 必填：`来源卡牌`。
- 不需要：`部署地区`、`目标卡牌`。
- 约束：每回合通常只能执行一次；仅当前行动玩家可执行；如卡牌带“非资产”限制会被拒绝。

### `play_card`（打出卡牌）
- 用途：从手牌打出角色/附属/事务。
- 必填：`来源卡牌`。
- 角色：通常还需要 `部署地区`。
- 角色明置（`face_up`）：按牌面费用结算，并检查忠诚需求。
- 角色暗置（`face_down`，秘密派遣）：固定消耗 `1` 资源，不检查忠诚需求。
- 附属：通常还需要 `目标卡牌`（宿主）。
- 角色/附属：仅当前行动玩家可打出。
- 附属/事务：费用与忠诚由服务端强校验，不满足会返回拒绝原因。

### `declare_attack`（发起攻击）
- 用途：本方驻场角色攻击目标角色。
- 必填：`来源卡牌` + `目标卡牌`。

### `declare_investigation`（发起调查）
- 用途：本方驻场角色对目标地区进行调查/放置影响。
- 必填：`来源卡牌` + `目标卡牌`（地区）。

### `move_card`（移动卡牌）
- 用途：将驻场单位移动到目标地区。
- 必填：`来源卡牌` + `目标卡牌`（地区）。

### `set_marker` / `remove_marker`（玩家标志）
- 用途：对玩家级标志进行增减。
- 必填：`目标玩家` + `标记类型` + `标记数量`。

### `set_card_marker` / `remove_card_marker`（卡牌标记）
- 用途：对卡牌级标记进行增减。
- 必填：`目标卡牌` + `标记类型` + `标记数量`。

## 常见排错
- 返回 `ActionRejected`：优先看 `reasonCode / messageKey / context`。
- 结束行动后无变化：看信息日志中是否出现 `step_ended` 与后续 `phase_advanced`。
- 对局结束后按钮禁用：属于正常行为，需 `重置并返回开局设置` 后开始新局。
