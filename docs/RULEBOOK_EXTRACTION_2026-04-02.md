## 2. 游戏骨架规则（回合与优先权）

### 2.1 回合结构

- 每回合固定四阶段：开始阶段 -> 行动阶段 -> 对抗阶段 -> 结束阶段。
- 开始阶段：重置步骤 -> 准备步骤 -> 抓牌步骤。
- 行动阶段：先手行动步骤 -> 后手行动步骤。
- 对抗阶段：按地区顺序 `地区1 -> 地区2 -> 地区3`，每个地区按 `调查 -> 战斗 -> 势力` 顺序结算。
- 结束阶段：结束步骤 -> 恢复步骤。

### 2.2 行动权与堆叠

- 只有具有行动权的玩家能行动。
- 阶段/步骤开始时，通常由先手获得行动权；唯一例外是后手行动步骤，由后手先获得行动权，且该步骤每次结算后仍由后手先拿行动权。
- 玩家可在行动权下选择发动快速行动或让过。
- 双方连续让过时：若堆叠非空，结算堆叠顶端对象；若堆叠为空，当前步骤/阶段结束。
- 每次堆叠对象结算后，行动权重新分配（通常先手先拿，后手行动步骤例外）。

### 2.3 行动分类

- 标准行动：仅可在自己行动步骤、且堆叠为空、且自己有行动权时执行。
- 快速行动：可在任意阶段发动，也可响应其他对象。
- 标准行动四类：建立资产（每回合通常 1 次）、打出卡牌、秘密派遣（每回合通常 1 次）、发动行动能力。

### 2.4 响应许可与禁止

可被响应：
- 打出卡牌
- 秘密派遣
- 付费令暗藏者现身
- 发动行动能力
- 发动触发能力

不可被响应：
- 建立资产
- 持续能力
- 令玩家获得费用的行动（含横置资产得费）
- 造成伤害本身

## 3. 卡牌与区域状态模型

### 3.1 区域与在场

- 区域：手牌、玩家牌库、世界牌库、争夺区、墓地、资产区、计分区、秘社。
- “场上”包含：争夺区、资产区、秘社、争夺中的地区牌。
- 堆叠不属于场上。

### 3.2 拥有者与操控者

- 拥有者自始至终不变。
- 卡牌进场后默认由拥有者操控，除非效果改写。
- 结附在对手牌上的附属仍由附属拥有方操控。
- “你/本方/敌方/友方”等指向以当前操控关系为准（友方用于多人规则语境）。

### 3.3 进场、离场、翻面

- 进场：打出结算、作为资产放置、暗藏者翻面成为角色、由效果从其他区域放置进场。
- 离场：场上牌移到其他区域（回手、牌库底、墓地等）；角色翻面成暗藏者也算离场。
- 进入地区的合法路径：正面打出、暗藏者现身、移动、效果直接放入。

### 3.4 卡牌类别与状态替换

- 基础类别：地区、角色、附属、事务。
- 特殊在场状态会替换类别语义：资产、暗藏者。
- 资产化后仅保留派系/魔法领域图标；原费用、忠诚、类别、防御力、文本能力等不再生效。
- 暗藏者背面在场，不具有正面信息，不受“影响角色”的效果影响，也不受伤害。

### 3.5 附属与封印

- 附属结算后必须结附合法对象；目标非法则附属进墓地。
- 宿主离场/翻面或不再满足结附条件时，附属消灭。
- 封印牌不在场上、牌面视为空白、不可被指定；场上牌被封印视为离场并移除其指示物。
- 承载封印的宿主离场时，封印牌回其拥有者手牌。
- 若封印结算时目标失效，待封印牌进其拥有者墓地。

### 3.6 指示物与状态动作

- 指示物含势力标志、时间标志、锁定标志、能量标志等。
- 角色/附属上的指示物也视作位于其所在地区/宿主对象上。
- 场上牌离场或翻面时移除其上所有指示物。
- 同一地区双方势力标志并存时，立即移除相等数量，直到仅剩一方（状态动作，持续检查，不入栈）。

## 4. 费用、忠诚与目标合法性

- 费用通过横置资产获得，通常每横置 1 资产得 1 费用。
- 每个步骤结束清空未使用费用，不可跨步骤累积。
- 打出/发动流程中，目标选择与“选择一项”均在支付前完成，支付后不得更改。
- 费用包含 `X` 时，支付前确定 `X` 数值。
- 忠诚需求在打出或暗藏者翻面时检查；需求可为派系图标或魔法领域图标。
- 否定性语句（“不能”）优先于允许性语句。
- 结算时若全部目标非法，则该对象不生效且不退款。

## 5. 三类对抗与奖励结算

### 5.1 共同规则

- 调查/战斗/势力对抗都只统计对应图标且未横置角色。
- 图标多者获胜；图标相等为平局；双方均 0 则该次对抗未发生。
- 平局时先手可发动“先手标志特权”：支付 1 费用并展示先手标志，把平局改为先手以 `+1` 优势获胜。
- 先手标志特权每回合限一次，不入栈，不可响应。

### 5.2 调查奖励

- 奖励值 `X = 己方调查图标 - 对方调查图标`。
- 获胜方检视牌库顶 `X` 张，任意分配回牌库顶/底顺序后，再抓 1 张。
- 获得调查奖励为必须执行，不入栈。

### 5.3 战斗奖励

- 奖励值 `X = 己方战斗图标 - 对方战斗图标`。
- 获胜方把 `X` 点战斗伤害分配给该地区敌方角色；若存在合法敌方角色则必须分配。
- 伤害分配完成后同时造成。
- 杀伤 `+N` 在该时机追加可分配战斗伤害。
- 获得战斗奖励不入栈。

### 5.4 势力奖励

- 奖励值 `X = 己方势力图标 - 对方势力图标`。
- 获胜方在该地区放置 `X` 个本方势力标志。
- 获得势力奖励为必须执行，不入栈。

### 5.5 赢取地区流程

当满足“堆叠为空 + 某方势力标志数 >= 地区势力值上限”时：

1. 将该地区的暗藏者/角色/附属置于各自拥有者牌库底（顺序由拥有者决定）。
2. 将该地区牌放入获胜者计分区。
3. 从世界牌库翻开 1 张地区补空位，维持 3 张争夺地区。
4. 地区赢取触发进入堆叠。
5. 先手获得行动权，可继续快速行动。

## 6. 关键能力与关键词抽出

- 触发能力：满足时机可发动；“强制”触发必须发动；多个同时触发时按先手组先入栈、再后手组入栈，各自决定组内顺序。
- 行动能力：主动发动，通常 `费用:效果`。
- 持续能力：在场即持续生效，不入栈。
- 公开：不能以秘密派遣方式进场。
- 隐匿：只能以秘密派遣方式进场。
- 机动：对抗阶段开始时可移动，单角色每回合至多触发一次。
- 袭击 X：现身时对同地区目标角色造成 X 伤害。
- 杀伤 X：战斗获胜时增加可分配战斗伤害 X。
- 威名：战斗对抗获胜后可放 1 势力标志，不叠加。
- 声望：地区三类对抗结束后可放 1 势力标志，不叠加。
- 灵体：分配到伤害时若己方对应魔法领域数量优势，则防止该伤害。
- 撤回：地区被赢取时该角色回手而非去牌库底。
- 领袖：同一玩家同时最多 1 个领袖在场。
- 独有：同名独有牌同一操控方同时最多 1 张在场。
- 标志 X：进场触发，放置 X 个指定指示物。
- 遏制 X：选择玩家移除其场上 X 个势力标志（不足则全移）。

## 7. 结束阶段与胜负判定

- 恢复步骤先按先手后后手依次执行“超手牌上限弃牌”（默认上限 7，受卡牌效果修正）。
- 然后同时移除角色伤害，并终止“本回合中/直到回合结束”效果。
- 若恢复步骤触发能力入栈，则处理完再次进入恢复流程，直到无触发。
- 最后传递先手标志，回合结束。

胜负：
- 任意时刻分数 >= 8 立即获胜。
- 若双方同时 >= 8，比高分；若同分则继续游戏。
- 任意时刻将要抓牌但牌库为空则该玩家立即失败。
- 抓牌步骤若双方同时需要抓牌且牌库都为空，则平局。

## 8. 可直接转引擎断言的规则清单

- `ASSERT_STACK_LIFO`: 堆叠后进先出。
- `ASSERT_PRIORITY_PASS_RULE`: 双方连续让过 + 空堆叠 => 结束当前步骤/阶段。
- `ASSERT_STANDARD_ACTION_GATE`: 标准行动需“自己行动步骤 + 有行动权 + 空堆叠”。
- `ASSERT_BUILD_ASSET_NONRESPONDABLE`: 建立资产不入栈不可响应。
- `ASSERT_SECRET_DEPLOY_RESPONDABLE`: 秘密派遣入栈可响应。
- `ASSERT_TARGET_LOCK_BEFORE_PAYMENT`: 目标与选项在支付前锁定。
- `ASSERT_ILLEGAL_TARGET_FIZZLE`: 结算时目标非法 => 效果失效且不退款。
- `ASSERT_FORCE_MARKER_CANCEL_SBA`: 双方势力标志并存立即等量抵消（状态动作）。
- `ASSERT_REGION_WIN_ONLY_ON_EMPTY_STACK`: 赢区检查要求堆叠为空。
- `ASSERT_REGION_WIN_DISPOSAL_ORDER`: 先清该地区单位再移地区牌再补新地区。
- `ASSERT_RECOVERY_DISCARD_ORDER`: 恢复步骤先手后后手依次弃到上限。
- `ASSERT_ENDSTEP_CLEANUP_SIMULTANEOUS`: 清伤与结束回合效果同时发生。
- `ASSERT_FIRST_PLAYER_PRIVILEGE_LIMIT`: 先手标志特权每回合限一次。
- `ASSERT_DRAW_FROM_EMPTY_LOSE`: 抓空牌库即失败（抓牌步骤双空例外平局）。

## 11. 模糊点裁定附录（含《霸权说明书》与实局示例旁证）

### 11.1 裁定优先级

当不同文本来源表述不一致时，按以下优先级处理：

1. 《规则手册》“阶段流程事件详解”及其流程子条款（编号规则）。
2. 《规则手册》术语词条 + 《玩家指南》对应规则段落 + 《霸权说明书》非扩展通用章节（第 3/7/8/9/10/15 章）。
3. “对抗阶段示例/实局示例”（用于验证流程直觉，不直接改写硬规则）。

### 11.2 冲突条目与当前裁定

`C01 抓牌步骤是否可以发动快速行动`

- 冲突来源：
- 术语词条“抓牌步骤”写成“该步骤中不能发动快速行动”。
- 详细流程 `1.3.3`、玩家指南流程、霸权说明书流程都写成“抓牌后可进入快速行动窗口”。
- 当前裁定：
- 采用可快速行动口径：`抓牌（不入栈） -> 抓牌触发入栈 -> 行动权窗口`。

`C02 恢复步骤中“超手牌上限弃牌”与“清伤/回合结束效果终止”的先后顺序`

- 冲突来源：
- 术语词条、玩家指南摘要、回合速查倾向“先清伤与结束回合效果，再弃牌”。
- 详细流程 `4.2.1-4.2.3` 明确“先手弃牌 -> 后手弃牌 -> 同时清伤与结束回合效果”。
- 当前裁定：
- 引擎抽出按详细流程执行（`4.2.1-4.2.3`）；
- 该项保留“待实局回放确认”标记，若后续拿到结束阶段实录，以实录和纸规复核后再最终固化。

`C03 赢取地区流程中“清场”与“地区入计分区”的顺序`

- 冲突来源：
- 术语词条与玩家指南写成“先地区入计分区，再处理该地区单位”。
- 详细流程 `赢取地区牌的流程 1.2.1-1.2.2` 写成“先处理该地区单位，再地区入计分区”。
- 当前裁定：
- 当前抽出按详细流程优先（先清场后入计分区）；
- 文档层面保留冲突注记，后续可在对局录像/纸规再校时复核。

`C04 秘密派遣是否可被响应`

- 冲突来源：
- 规则手册与玩家指南均将秘密派遣列为可响应动作。
- 霸权说明书通用章节出现“秘密派遣不可响应”表述，与前两者冲突。
- 当前裁定：
- 采用“秘密派遣可被响应”口径；
- 将霸权该处视作编排/转写误差，不用于覆盖主规则。

`C05 双方图标都为 0 的对抗是否属于平局`

- 证据：
- 详细流程三类对抗都明确“双方均为 0 -> 对抗未发生”。
- 玩家指南与示例也按“跳过/未发生”处理。
- 当前裁定：
- “未发生”不等于平局，不能触发先手标志特权。

`C06 先手标志特权的性质`

- 证据：
- 规则手册与霸权说明书都明确：每回合一次、不入栈、不可响应、用于平局改判。
- 当前裁定：
- 作为特殊规则事件实现，不建模为普通行动能力对象。

### 11.3 实局示例的使用边界

- 示例可用于确认“每地区三类对抗顺序”“对抗前后存在响应窗口”“暗藏者可在关键时点现身并改变后续对抗结果”。
- 示例不作为数值硬规则的唯一依据：示例中可能叠加了未完整展开的卡牌能力，不能反推“基础奖励公式”。
